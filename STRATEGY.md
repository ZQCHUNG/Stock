# 交易策略說明

## 策略 v1（基礎版）— 六指標加權評分

### 概述
使用六大技術指標各自獨立評分（-1 ~ +1），加權後得到綜合分數，超過閾值觸發買賣。

### 指標與評分邏輯

#### 1. MA 移動平均線（權重 20%）
- MA5 > MA20：+0.5（短期趨勢向上）
- MA20 > MA60：+0.5（中長期趨勢向上）
- 反之各 -0.5
- **多頭排列（MA5>MA20>MA60）= +1，空頭排列 = -1**

#### 2. RSI 相對強弱指標（權重 20%）
- RSI < 30：+1（超賣反彈機會）
- RSI 30~50：線性插值 +1 → 0
- RSI 50~70：線性插值 0 → -1
- RSI > 70：-1（超買回檔風險）

#### 3. MACD（權重 20%）
- MACD > Signal 線：+0.5
- MACD 柱狀體 > 0：+0.5
- 反之各 -0.5

#### 4. KD 隨機指標（權重 20%）
- K > D：+0.5（黃金交叉）
- K < 20：+0.5（超賣區）
- K < D：-0.5（死亡交叉）
- K > 80：-0.5（超買區）

#### 5. 布林通道（權重 10%）
- 股價位置 = (收盤 - 下軌) / (上軌 - 下軌)
- 分數 = 1 - 2 * 位置（靠下軌偏多，靠上軌偏空）

#### 6. 成交量（權重 10%）
- 量增價漲：+0.5（健康上漲）
- 量縮價跌：+0.3（洗盤結束可能）
- 量增價跌：-1.0（出貨訊號）
- 量縮價漲：0.0（動能不足）

### 訊號產生
- 綜合評分 > 0.3 → **BUY（買入）**
- 綜合評分 < -0.3 → **SELL（賣出）**
- 其他 → **HOLD（持有/觀望）**

### 交易規則
- 買入：用全部可用資金，以「張」為單位（1000 股）
- 賣出：全部出清
- 一次只持有一檔（單一股票回測）

### 費用
- 手續費：0.1425%（買賣各收一次）
- 交易稅：0.3%（賣出時收）

---

## v1 已知問題

1. **沒有停損** — 虧損不設限，等到賣出訊號才出場
2. **沒有停利** — 不鎖利潤，大漲後可能反轉倒賠
3. **勝率偏低** — 指標交叉產生大量假訊號
4. **全押策略** — 每次 all-in 沒有資金管理
5. **無趨勢過濾** — 盤整期也頻繁進出，白交手續費

## 策略 v2（改進版）— 已實作

> v2 針對 v1 的五大問題逐一改善，所有參數可在 Web 介面側邊欄即時調整。

### 1. 停損停利（backtest/engine.py、simulation/simulator.py）

| 機制 | 預設值 | 說明 |
|------|--------|------|
| 固定停損 | -7% | 收盤價跌破買入價 × 0.93 → 強制出場 |
| 移動停利 | -5% | 持倉期間最高價回落 5% → 出場鎖利 |

- 每日以**當日最高價**更新 `highest_since_entry`
- 停損停利**優先於訊號**，即使當天訊號是 HOLD 也會觸發
- 交易紀錄會標記出場原因：`停損` / `移動停利` / `訊號賣出` / `期末平倉`

### 2. 訊號確認機制（analysis/strategy.py）

| 機制 | 預設值 | 說明 |
|------|--------|------|
| 連續確認天數 | 2 天 | `raw_signal` 需連續 N 天為 BUY 才確認進場 |
| 量能確認 | 開啟 | 買入當天成交量需 > 5 日均量 |

- 用 `rolling(window=N).sum()` 計算連續 BUY 天數
- 未達確認條件的 BUY 會被降級為 HOLD
- `raw_signal` 保留原始訊號，`signal` 為過濾後最終訊號

### 3. 趨勢過濾（analysis/strategy.py）

| 機制 | 預設值 | 說明 |
|------|--------|------|
| 趨勢過濾 | 開啟 | MA20 < MA60（空頭排列）時不產生 BUY |

- 避免在中期下跌趨勢中逆勢做多
- 只影響 BUY 訊號，SELL 不受影響

### 4. 資金管理（backtest/engine.py、simulation/simulator.py）

| 機制 | 預設值 | 說明 |
|------|--------|------|
| 單筆最大部位 | 50% | 每次買入最多用可用資金的 50% |

- v1 每次 all-in（100%），v2 保留現金緩衝
- 降低單筆交易對整體權益的衝擊

### 5. 可調參數一覽（config.py）

```python
RISK_PARAMS = {
    "stop_loss": 0.07,          # 停損百分比
    "trailing_stop": 0.05,      # 移動停利百分比
    "max_position_pct": 0.5,    # 單筆最大部位
}

STRATEGY_PARAMS = {
    ...
    "confirm_days": 2,          # 訊號確認天數
    "trend_filter": True,       # 趨勢過濾開關
    "volume_confirm": True,     # 量能確認開關
}
```

### v2 vs v1 改善重點

| 問題 | v1 | v2 |
|------|-----|-----|
| 停損 | 無 | 跌 7% 強制出場 |
| 停利 | 無 | 從高點回落 5% 出場 |
| 假訊號 | 多 | 需連續 2 天確認 + 量能確認 |
| 逆勢交易 | 有 | MA20>MA60 才做多 |
| 資金管理 | 全押 | 單筆最多 50% |

## 策略 v3（實驗版）— 已實作

> v3 嘗試用 ATR（平均真實波幅）動態調整停損停利，取代固定百分比。
> 經 30 隻隨機股票回測驗證，ATR 停利在部分個股有幫助，但整體不如 v2 的固定 5%。
> 因此 **ATR 預設關閉**，可在 UI 手動開啟嘗試。

### v3 新增功能

#### 1. ATR 動態停損停利（可選）

| 機制 | 預設值 | 說明 |
|------|--------|------|
| ATR 停損倍數 | 3.0x | 停損距離 = 進場時 ATR × 3.0 |
| ATR 停利倍數 | 2.5x | 停利距離 = 進場時 ATR × 2.5 |
| ATR 開關 | **關閉** | 預設用固定百分比（v2），可手動開啟 |

- ATR 用 14 日 EMA 計算（見 `indicators.py`）
- 進場時鎖定 ATR 值，整筆交易不變
- 波動大的股票自動放寬停利，小波動股票自動收緊

#### 2. 最短持有天數

| 機制 | 預設值 | 說明 |
|------|--------|------|
| min_hold_days | 2 天 | 進場後 2 天內不觸發停損停利 |

- 避免正常波動在進場隔天就被掃出去
- 給交易足夠時間發展

#### 3. 評分趨勢確認（可選，預設關閉）

| 機制 | 預設值 | 說明 |
|------|--------|------|
| score_rising | **關閉** | 要求 composite_score 比前一天高才進場 |

- 實測過於嚴格，大幅減少進場機會（30 隻中 11 隻無交易）
- 預設關閉，有需要可手動開啟

### 30 隻隨機股票回測實驗結果

| 指標 | v2 (固定5%) | v3a (ATR無上限) | v3 (ATR+cap 8%) |
|------|:-:|:-:|:-:|
| 獲利股票 | **9/30** | 8/30 | 7/30 |
| 平均報酬 | **-0.05%** | -0.56% | -0.60% |
| 平均回撤 | **-3.48%** | -4.79% | -4.52% |
| 勝率 | 28.1% | **32.0%** | **32.0%** |
| 移動停利出場 | 86% | 74% | 78% |

**結論：v2 的固定 5% 停利整體最穩。** ATR 放寬停利會讓虧損擴大。
策略的核心瓶頸是 **進場品質（28% 勝率）**，而非出場方式。

## 策略 v4（趨勢動量版）— 已實作 ⭐ 推薦

> v4 完全重構進出場邏輯，捨棄 v1-v3 的評分系統，改用趨勢動量進場 + 移動停利出場。
> 經 30 隻隨機股票 3 年回測驗證，**平均報酬 +87.7%、勝率 63.1%、87% 股票獲利**。

### 核心設計理念

v4 的關鍵改進來自三個發現：

1. **進場品質**：只在「確認的上升趨勢中」買入，用 ADX + DI 方向確認趨勢存在
2. **出場效率**：移動停利（trailing stop）讓獲利持續奔跑，不像固定 TP 過早離場
3. **持有耐心**：最短持有 5 天，避免正常波動觸發假停損

### 進場邏輯（analysis/strategy_v4.py）

#### 進場前提條件（全部必須滿足）

| 條件 | 參數 | 說明 |
|------|------|------|
| 上升趨勢 | min_uptrend_days = 10 | MA20 > MA60 至少持續 10 天 |
| 趨勢強度 | adx_min = 18 | ADX ≥ 18（有趨勢） |
| 方向確認 | +DI > -DI | 上升方向確認 |
| RSI 範圍 | 30 ~ 80 | 排除超賣超買極端值 |
| 量能確認 | min_volume_ratio = 0.7 | 成交量 ≥ 5 日均量的 70% |

#### 進場模式（滿足前提後，二選一觸發）

| 模式 | 條件 | 說明 |
|------|------|------|
| **支撐反彈** | 價格在 MA20 附近（-2% ~ +5%），且今日收盤 > 昨日 | 在趨勢中的拉回支撐處買入 |
| **動量延續** | MA5 > MA20 > MA60，且 ROC > 2 | 在強多頭排列中順勢買入 |

### 出場邏輯（backtest/engine.py run_v4）

| 機制 | 參數 | 說明 |
|------|------|------|
| 固定停利 | +10% | 從買入價上漲 10% 出場 |
| 固定停損 | -7% | 從買入價下跌 7% 出場 |
| 移動停利 | 2% | 從持倉期間最高價回落 2% 出場 |
| 最短持有 | 5 天 | 進場後 5 天內不檢查 TP/SL |

**出場偵測方式**：用當日最高價（high）偵測停利、最低價（low）偵測停損，更貼近真實盤中行為。

#### 移動停利運作方式

```
進場價 100 元，初始停損 = 93 元（-7%）
第 6 天：最高 105 元 → 移動停損提升至 102.9 元（105 × 0.98）
第 8 天：最高 112 元 → 移動停損提升至 109.8 元（112 × 0.98）
第 9 天：最低跌到 109 元 → 觸發移動停利出場，獲利 +9.8%
```

### 新增技術指標

| 指標 | 計算方式 | 用途 |
|------|----------|------|
| ADX (14) | Wilder 平滑法 | 趨勢強度（>18 有趨勢） |
| +DI / -DI | 方向指標 | 上升/下降方向 |
| ROC (12) | 12 日收盤價變化率 | 動量衡量 |

### 可調參數（config.py）

```python
STRATEGY_V4_PARAMS = {
    "adx_min": 18,              # ADX 最低要求
    "rsi_low": 30,              # RSI 下限
    "rsi_high": 80,             # RSI 上限
    "min_uptrend_days": 10,     # 上升趨勢最少天數
    "support_max_dist": 0.05,   # 支撐進場最大距離 MA20
    "min_volume_ratio": 0.7,    # 最低量能比
    "take_profit_pct": 0.10,    # 停利 +10%
    "stop_loss_pct": 0.07,      # 停損 -7%
    "trailing_stop_pct": 0.02,  # 移動停利 2%
    "min_hold_days": 5,         # 最短持有天數
    "max_position_pct": 0.9,    # 單筆最大部位 90%
}
```

### 30 隻隨機股票回測結果（3 年期）

| 指標 | v2 (評分系統) | v4 (趨勢動量) |
|------|:-:|:-:|
| 獲利股票 | 9/30 (30%) | **26/30 (87%)** |
| 平均報酬 | -0.05% | **+87.7%** |
| 中位數報酬 | — | **+48.8%** |
| 平均勝率 | 28.1% | **63.1%** |
| 平均回撤 | -3.5% | -12.4% |
| 平均 Sharpe | — | **0.70** |
| 平均交易次數 | ~3/股 | **22/股** |

### v4 vs v1-v3 差異

| 項目 | v1-v3 | v4 |
|------|-------|-----|
| 進場 | 六指標加權評分 > 閾值 | 趨勢確認 + 支撐/動量模式 |
| 出場 | 評分 < 閾值 / 固定停損停利 | 移動停利 + 固定 TP/SL |
| 訊號類型 | BUY / SELL / HOLD | BUY / HOLD（出場由 TP/SL 處理） |
| 趨勢判斷 | MA 交叉 + RSI + KD | ADX + DI 方向 + 上升天數 |
| 量能分析 | 量價關係評分 | 量能最低門檻過濾 |

### 迭代過程

v4 經過 14 輪 prototype 迭代測試：

1. **v4a**：基礎版固定 TP10/SL5，+2.2%，36% WR
2. **v5-v7**：測試拉回/突破/雙 MA 進場模式
3. **v8**：發現 3 年數據顯著優於 2 年（+8.3% vs -2.0%）
4. **v9-v10**：組合模式 + 90% 部位，+17.1%，42% WR
5. **v12**：發現 high/low 偵測 TP/SL + trailing stop 大幅改善
6. **v13-v14**：微調 min_hold 5 天 + trailing 2%，達成最終成績
